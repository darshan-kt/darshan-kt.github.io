Autonomous Outdoor Navigation Robot

Robust RTK-Based Navigation for Dynamic Environments

Overview: This project details the development of a large-scale outdoor navigation robot featuring inflatable tyres and a lightweight aluminium extrusion frame. Designed in Fusion 360, the robot integrates advanced sensor suites and a robust RTK/GPS setup, enabling precise localization and autonomous navigation in challenging outdoor environments.

Abstract

This project focuses on the development of an autonomous outdoor navigation robot engineered for operation in GPS-restricted and dynamically changing environments. The system utilizes a Raspberry Pi 4 as the central processor, integrated with an RPlidar S2 for outdoor ranging, and a Bosch BNO055 IMU for inertial measurements. A high-precision RTK setup, incorporating Ardusimple RTK2B boards and an innovative NTRIP configuration via Rtk2go.com, provides centimeter-level localization accuracy. This report details the design, sensor integration, and software architecture that enable robust autonomous navigation and precise waypoint tracking.

Introduction

In modern outdoor robotics, accurate localization is essential for safe and efficient navigation. This project addresses common challenges—such as sensor noise, odometry drift, and environmental interference—by integrating an RTK-based GPS system with conventional sensor inputs. The robot's mechanical design, featuring inflatable tyres and a chassis constructed from aluminium extrusions, ensures high mobility and durability in rugged terrain. Using a Raspberry Pi 4 running ROS, advanced mapping and navigation algorithms are deployed to enable autonomous operation.

Base Station Setup

A key innovation of the system is the remote RTK base station setup. A dedicated GPS board, equipped with a high-gain base station antenna, is connected to a telemetry module via UART. This module continuously transmits correction data. To relay these corrections over the internet, a Raspberry Pi Zero W is deployed at the remote base station, where it receives telemetry signals through an attached receiver module. The Pi Zero W then uploads the RTCM correction data to an NTRIP server hosted on Rtk2go.com.

At the robot end, the Raspberry Pi 4, with internet access, downloads the RTCM data from the NTRIP server and feeds it into the GPS module via USB. The RTK2B board processes these corrections internally and outputs highly accurate positional data via a second USB port.

Robot Setup and Software Integration

The outdoor navigation stack is built on the ROS framework. The Raspberry Pi 4 serves as the central processing unit, integrating data from the RPlidar S2 and Bosch BNO055 IMU to construct a real-time map of the environment. The RTK-based GPS corrections are fused with the lidar and inertial data via the robot_localization package, ensuring robust and precise localization.

Waypoints are defined in Mapviz and are fed to the ROS navigation stack. The move_base node, in conjunction with the TEB planner, autonomously guides the robot toward the goal points, even in the presence of dynamic obstacles and environmental uncertainties.

Experimental Validation

Field tests demonstrated the effectiveness of the integrated RTK system, where the robot consistently reached its designated waypoints with centimeter-level accuracy. The fusion of RTK corrections with lidar and IMU data significantly reduced odometry errors, enabling reliable autonomous navigation in challenging outdoor environments.

Conclusion

The developed autonomous outdoor navigation robot exemplifies the integration of low-cost hardware with advanced RTK and ROS-based navigation techniques. The innovative base station setup and robust sensor fusion ensure high-accuracy localization and reliable performance in dynamic, real-world conditions. This work paves the way for further improvements in outdoor robotic systems and has significant implications for applications in surveillance, delivery, and remote exploration.

Future Scope

  • Integration of additional vision-based sensors for enhanced environmental perception.
  • Optimization of the sensor fusion algorithm to further reduce latency and improve accuracy.
  • Expansion of the navigation stack to support more complex mission planning and autonomous decision-making.
  • Scaling up the system for industrial or commercial applications in harsh outdoor environments.
